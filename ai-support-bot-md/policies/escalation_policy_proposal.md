# Policy 改訂提案: escalation_policy (03_escalation_policy.md)

**変更理由:**  
500件分析で判明した課題:
1. 解約示唆が「即時 escalation」に分類されているが、定型フォーム形式の解約申請13件は structured handoff の方が担当者対応効率が高い
2. `experience_issue`（新設）に対応したエスカレーション条件を追加する必要がある
3. NocoDB の severity フィールドが全件 NULL のため、重大度は raw_body の LLM 判定に依存している実態を明記する

---

## 変更箇所

### 1. 即時エスカレーション条件の「解約示唆」の精緻化

**変更前:**
> 強いクレーム、解約示唆

**変更後:**
> 強いクレーム（怒り・放置への不満・複数回要求）  
> 解約に伴う返金要求  
> **解約申請そのもの（フォーム形式・定型申請）は escalation 不要 → billing_contract intent で handoff**

**理由:** 解約申請は「契約終了の事務手続き」であり、escalation の目的である「緊急対応・補償判断・感情対応」には該当しない。定型申請を escalation すると担当者が対応ステータスを管理しにくくなる。

### 2. experience_issue に対応したエスカレーション条件の追加

**追加:**
```
体験/WEB接客 関連の即時エスカレーション条件:
  - 「本番サイトで体験が全面停止している」「全ユーザーに影響している」→ 即時 escalation
  - 「至急」「本日リリース予定」「本番で止まっている」 → 即時 escalation
  - 広告施策・EC 購買に重大影響を与えている体験バグの示唆 → escalation 検討
```

### 3. NocoDB severity フィールド未入力の実態への対応

**追加:**
```
重大度判定の実態:
  - NocoDB の severity フィールドは現在自動入力されていない
  - 重大度判定は LLM が raw_body から判断する必要がある
  - 将来的には LLM 分類時に severity を NocoDB に自動書き込みする実装を追加する（推奨）
```

### 4. Botが2ターン質問してもカテゴリ確定できない場合のルール見直し

**変更前:**
> Botが2ターン以上質問してもカテゴリ確定できない場合

**変更後:**
> Botが**3ターン以上**質問してもカテゴリ確定できない場合

**理由:** 500件分析で顧客発話が4ターン以上のケースが46%存在する。2ターンでの escalation は早すぎる。intent が特定できなくても、slot 収集が進んでいれば handoff で十分。

## 変更後の全文（参考）

```markdown
# Escalation Policy

以下に該当する場合は、追加質問を最小限にして人へエスカレーションする。

## 即時エスカレーション条件

- セキュリティ、個人情報、権限漏洩
- 全面障害の可能性（全サービス・全ユーザー影響）
- 「至急」「緊急」「全員」「全く使えない」等の高緊急ワード
- エンタープライズ重要顧客の重大不具合
- 契約補償・返金・法務判断
- **強いクレーム（怒り・放置への不満・複数回の同一要求）**
- **解約に伴う返金要求**（解約申請そのものは escalation 不要）
- 体験/WEB接客が本番サイトで全面停止している
- 広告施策・EC 購買への重大ビジネス影響の示唆
- Botが **3ターン以上** 質問してもカテゴリ確定できない場合

## 解約申請の特別ルール（escalation 除外）

以下をすべて満たす場合は escalation 不要（billing_contract handoff で対応）:
- inquiry_topic に「解約」が含まれる
- account_email_or_user が取得済み（フォーム形式では通常含まれる）
- 怒り・クレーム・返金要求のシグナルがない

## エスカレーション時の動作

- 現時点までの取得情報を構造化要約する
- 未取得項目を明記する
- 追加推測は書かない

## 重大度判定の注意

NocoDB の severity フィールドは現在自動入力されていない。  
重大度は LLM が raw_body から判断すること。  
将来的には LLM 分類時に severity を NocoDB に自動書き込みする実装を追加する（推奨）。
```
